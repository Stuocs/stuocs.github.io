Recruit is a CTF that challenges the user to bypass security protocols by mapping the structure Human Resources web app and get access as administrator inside the web app.


![hosts_file](../../../../../assets/img/THM/Recruit/Pasted%20image%2020260429135844.png)

In first place, as the image above, to avoid confusion we will add the IP inside /etc/hosts by using `sudo vim /etc/hosts` and we will tell the machine that the IP that we are going to attack is going to be named under `recruit.local`.

![nmap](../../../../../assets/img/THM/Recruit/Pasted%20image%2020260429141801.png)

As is highly recomended, an nmap scan can be done but in 3 different stages.
1. Is important to identify all the open ports so for that is viable to use the following command:
   ```shell
   sudo nmap -p- --min-rate 5000 -Pn -T4 <objective IP> -oN all_ports.txt
   ```
   If this one is executed as sudo or root it will have the argument `-sS` (Or stealthy scan) by default, otherwise it will use `-sT` (Connect scan).
2.  After that, once the open ports are known is recomendable to use the following command to identify the purpose of the ports:
   ```shell
   sudo nmap -p 22,53,80 -sC -sV --min-rate 5000 -Pn <objective IP> -oN filtered_ports.txt
   ```
3.  Finally, but not always needed, is possible to do exclusively a `-sU` (UDP scan) scan to identify UDP protocols, this is because, in contrast with TCP, UDP does not answer with a handshake so the usual way wont work on it:
   ```shell
   sudo nmap -sU --top-ports 20 -Pn recruit.local -oN udp.txt
   ```

```php

$query = "SELECT * FROM candidates WHERE name LIKE '%$search%'";
```

`%search`varible is not filtered in any way nor using a prepared statement leading to a vulnerability that probably allows to use a `UNION SELECT` to do the attack

It also prints `$sqlError` what basically is a mistake that happen to tell if it's really vulnerable or not.

2. There is no access control:
```php
if ($_SESSION['role'] === 'hr') {
    $flagPath = '/user.txt';
} elseif ($_SESSION['role'] === 'admin') {
    $flagPath = '/admin.txt';
}
```

The code does not check the role to do the search, just to approve or reject requests.
